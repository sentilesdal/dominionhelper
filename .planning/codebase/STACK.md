# Technology Stack

**Analysis Date:** 2026-03-17

## Languages

**Primary:**
- TypeScript 5.9.3 - Strict mode, ES2022 target. All source code in `src/` and `tests/`

**Secondary:**
- JavaScript (generated by build process) - IIFE bundles for Chrome extension
- HTML - Side panel UI (`src/sidepanel/sidepanel.html`)
- CSS - Side panel styling (`src/sidepanel/sidepanel.css`)
- JSON - Card database (`src/data/cards.json`) and configuration files

## Runtime

**Environment:**
- Node.js (no version specified in project; used for build scripts)
- Chrome 120+ (browser runtime for extension)
- Manifest V3 (Chrome Extension format)

**Execution Contexts:**
- ISOLATED world (content scripts) - `src/content/content.ts` injected into dominion.games
- MAIN world (game state bridge) - `src/content/game-state-bridge.ts` for Angular access
- Service Worker (background) - `src/background/service-worker.ts`
- Document (side panel) - `src/sidepanel/sidepanel.ts`

**Package Manager:**
- npm (package.json present)
- Lockfile: Not shown in exploration; likely present as `package-lock.json`

## Frameworks

**Core:**
- Chrome Extension APIs (Manifest V3) - Message passing, storage, side panel
- DOM APIs - MutationObserver for kingdom/log observation
- CustomEvents - Angular bridge communication between MAIN and ISOLATED worlds

**Testing:**
- Vitest 3.0.0 - Test runner and framework (`vitest.config.ts`)
- JSDOM 28.1.0 - DOM environment for unit tests

**Build/Dev:**
- Vite 7.3.1 - Module bundler with IIFE output for Chrome extension
- Custom Vite build script - `scripts/build.mjs` produces four bundles:
  1. `dist/content.js` - Kingdom analysis + deck tracker + card database
  2. `dist/service-worker.js` - Message hub and analysis engine caching
  3. `dist/sidepanel.js` - Side panel UI renderer
  4. `dist/game-state-bridge.js` - Angular game state polling

## Key Dependencies

**Critical:**
- Card database - Bundled from `src/data/cards.json` (786 cards, 18 Dominion sets)
- Chrome types - `@types/chrome` 0.1.37 for type-safe API access

**Infrastructure:**
- No external API dependencies - All analysis is client-side
- No database client - No backend integration
- No HTTP library - No network requests needed

## Configuration

**Environment:**
- No `.env` file needed - No secrets or API keys required
- `chrome.storage.session` used for caching current kingdom across service worker restarts
- Extension loads from `dist/` directory (built output)

**Build:**
- `tsconfig.json` - TypeScript compiler options (ES2022, strict, moduleResolution: bundler)
- `vite.config.ts` - Vitest configuration (test include pattern)
- `eslint.config.mjs` - ESLint flat config with TypeScript and Prettier integration
- `.prettierrc` - Prettier formatting (2 spaces, trailing commas, 80 char width, double quotes)
- `manifest.json` - Source manifest template; `dist/manifest.json` generated by build script

## Build Configuration Details

**Vite Builds:**
- Format: IIFE (immediately-invoked function expressions for Chrome)
- Minification: Disabled for debugging visibility
- Sourcemaps: Disabled (content scripts run in isolated world with limited devtools access)
- Path alias: `@` → `src/` directory
- No external dependencies - All imports are internal TypeScript modules
- JSON import (cards.json) bundled directly via `resolveJsonModule: true` in tsconfig

**Build Output:**
- Static assets copied: `icons/` directory, `sidepanel.html`, `sidepanel.css`
- Manifest transformation: Source manifest paths converted to `dist/` relative paths
- Final extension structure ready to load from `chrome://extensions` → "Load unpacked" → `dist/`

## Platform Requirements

**Development:**
- Node.js + npm for build and test commands
- Chrome browser with developer mode enabled
- TypeScript 5.9.3+ compiler (included in devDependencies)

**Production:**
- Chrome 120+ (or latest Chromium-based browser)
- Deployment: Load from `dist/` directory in `chrome://extensions/` (manual developer mode)
- No server-side deployment needed - Extension is entirely client-side

**Bundled Resources:**
- 786 card definitions from `src/data/cards.json` bundled into `dist/content.js`
- All analysis logic self-contained in extension bundles
- No external files or APIs required at runtime
